How It Works: Replayable Structural Analysis

Reflective Diagnostics is a forensic reconstruction pipeline. It operates in strict phases: Ingest (Probe), Analysis (L2), Synthesis (Interpreter), and Reporting.

Each phase operates on a frozen run artifact (inputs, hashes, structure, metrics), enabling exact replay across models and time.

1. Global Shell & Ingest

Run Context

Every interaction is bound to a unique Run ID (e.g., 2126FD14). This ensures strict separation of evidence between sessions.

Probe Console

  • Load Evidence: Analyze existing files
  • Run Live Probe: Inject prompts live
  • Input Types: Text, PDF, JSON

Uploaded files are indexed as external evidence and linked during claim–premise extraction.

Evidence Locker

Once capture completes, evidence is frozen. The locker generates SHA-256 hashes to prevent downstream alteration.

The anchor for replayability.

2. L2 Structural Analysis

Extraction Engine

Reconstructs model output into a claims–premises–conclusions support graph.

Telemetry Metrics

  • Coverage: % Substantiated
  • Orphans: Unsupported claims
  • Density: Reasoning interconnectivity
  • Stability: Consistency under load

Trace X-Ray

A highlighted view showing exactly where each claim and premise appears in the source text. No abstraction layer.

3. Interpreter & Reporting

Cross-Model Synthesis

Aggregates structural metrics across multiple models. Generates the Consensus Topology and Ranked Leaderboard.

Deterministic Copilots

  • Chart Copilot: Explains metrics
  • Interpreter: Recommends winner

Rule-based guidance only. No creative reasoning.

Court-Grade Report

Assembles locked artifacts, metrics, and definitions into a static PDF-style export for governance review.

Not a score — a case file.
