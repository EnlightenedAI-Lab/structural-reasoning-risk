How it works: a replayable measurement pipeline, not “LLM as judge.”

Reflective Diagnostics converts model outputs into an inspectable support structure (claims, premises, conclusions), computes structural signals, and produces an evidence packet that can be replayed and audited.

End to End Pipeline

INPUT → CONTROLLED RUN → ARTIFACT CAPTURE → STRUCTURAL EXTRACTION → SIGNAL COMPUTATION → INTERPRETER → EVIDENCE PACKET

Key property: each stage produces a fixed artifact that becomes the input to the next stage. No hidden rewrites. No silent retries.

1) Controlled Run

Purpose: comparability across models.

  • Same input, same constraints, across models
  • No hidden retries, no prompt rewriting, no adaptive steering
  • Model identity and version recorded
Why it matters: differences observed are attributable to the model, not shifting conditions.

2) Artifact Capture

Purpose: source of record.

  • Raw model outputs stored verbatim
  • Run metadata captured (timestamps, run ID, model selection)
  • Hashes generated to prevent silent alteration
Why it matters: the run can be audited by someone who did not witness it.

3) Structural Extraction

Purpose: convert free text into inspectable support structure.

  • Extract claims, premises, and conclusions
  • Link conclusions to premises (support chain)
  • Flag unsupported commitments (orphans)
Why it matters: the instrument evaluates support structure, not writing quality.

4) Signal Computation + Interpreter

Purpose: compute decision signals and compare models under identical constraints.

  • Compute structural signals (coverage, orphan rate, integrity, stability)
  • Cross model divergence: where models disagree under the same input
  • Interpreter produces a consolidated, inspectable summary
Why it matters: evaluators can see where conclusions are structurally justified (or not) and how consistent models are.

What This Is Not

  • Not truth verification
  • Not sentiment or policy merit scoring
  • Not a subjective “LLM judges LLM” rubric
This instrument measures derivability: whether conclusions follow from stated premises, with replayable evidence.
