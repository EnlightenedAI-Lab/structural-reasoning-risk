How it works: a replayable measurement pipeline for evaluating structural support in AI reasoning.

Reflective Diagnostics converts AI outputs into an inspectable support structure (claims, premises, conclusions), computes structural signals, and produces a replayable evidence record suitable for audit and external review.

End to End Measurement Flow

Outputs from each stage become fixed inputs to the next stage.
Input → Controlled Run → Artifact Capture → Structural Extraction → Signal Computation → Interpreter → Evidence Packet
No hidden rewrites. No silent retries. No adaptive steering.

1) Controlled Run (comparability)

Purpose: ensure differences are attributable to the model.

  • Same input, same constraints, across models
  • Model identity and version recorded
  • No hidden retries or prompt rewriting

Why it matters: observed differences reflect model behavior, not shifting conditions.

2) Artifact Capture (source of record)

Purpose: preserve evidence exactly as produced.

  • Raw model outputs stored verbatim
  • Run metadata captured (timestamps, run ID, model selection)
  • Hashes generated to prevent silent alteration

Result: a run can be audited by someone who did not witness it.

3) Structural Extraction

Purpose: convert free text into an inspectable structure.

  • Claims, premises, and conclusions explicitly identified
  • Support relationships made visible
  • Orphaned or unsupported conclusions exposed

This step transforms narrative output into analyzable structure.

4) Signal Computation

Purpose: quantify structural integrity from extracted structure.

  • Coverage
  • Orphan rate
  • Stability
  • Integrity

Signals are derived from structure, not preference or outcome grading.

5) Interpreter and Evidence Packet

Purpose: enable comparison, review, and exportable exhibits.

Interpreter
  • Cross model synthesis under identical constraints
  • Divergence and consistency surfaced
  • Failure modes presented with supporting artifacts
Evidence Packet
  • Replayable record of inputs, outputs, and computed signals
  • Exportable forensic exhibit format
  • Designed for audit, governance, and funding review
This is a measurement instrument: it produces inspectable structure and auditable artifacts.
