How it works: a replayable measurement pipeline — not “LLM as judge.”

The instrument converts model outputs into an inspectable support structure (claims → premises → conclusions), computes structural signals, and produces an evidence packet that can be replayed and audited.

End-to-End Measurement Flow

Input→ Controlled Run→ Artifact Capture→ Structural Extraction→ Signal Computation→ Interpreter→ Evidence Packet
Key rule: each stage produces a fixed artifact used by the next stage.
No: hidden rewrites · silent retries · post-hoc edits.

  A[Input\n(prompt / document / evidence)] --> B[Controlled Run\n(same constraints across models)]
  B --> C[Artifact Capture\n(raw output + metadata + hashes)]
  C --> D[Structural Extraction\n(claims / premises / conclusions)]
  D --> E[Signal Computation\n(coverage / orphans / integrity / stability)]
  E --> F[Interpreter\n(cross-model comparison + divergence)]
  F --> G[Evidence Packet\n(replayable record + exportable exhibit)]

1) Controlled Run

  • Same input, same constraints, across models
  • Model identity + version recorded
  • No adaptive steering or prompt rewriting

Purpose: comparability — differences are attributable to the model, not shifting conditions.

2) Artifact Capture

  • Raw outputs stored verbatim
  • Run metadata captured (timestamps, run ID, model selection)
  • Hashes generated to prevent silent alteration

Purpose: source of record — an evaluator can audit the run without witnessing it.

3) Structural Extraction

  • Convert free text into inspectable structure
  • Claims, premises, conclusions separated
  • Support links made explicit

Purpose: expose unsupported commitments and missing premises.

4) Signal Computation + Interpreter

  • Compute structural signals (coverage, orphans, integrity, stability)
  • Compare models under identical constraints
  • Generate an exportable evidence packet

Purpose: decision-grade inspection — structure and evidence, not opinions.
