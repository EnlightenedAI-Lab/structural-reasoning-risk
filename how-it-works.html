How the Instrument Works

This system is designed as a measurement instrument, not a chat interface or subjective evaluator. Every step is controlled, replayable, and produces artifacts suitable for audit and review.

End-to-End Pipeline

Input (prompt / document / evidence) → Multi-model execution (identical constraints) → Artifact capture (raw outputs, metadata, hashes) → Structural extraction (claims, premises, conclusions) → Metric computation (coverage, integrity, orphans, stability) → Interpreter synthesis (cross-model comparison) → Evidence locker (persistent, replayable case file)

Each arrow represents a non-skippable stage. Outputs from one stage become immutable inputs to the next.

Controlled Execution

  • Identical prompts across models
  • Explicit temperature and sampling controls
  • No hidden retries or prompt adaptation
  • Model identity and version captured

This ensures differences observed are structural, not artifacts of tuning or variance.

Determinism & Replay

  • Every run produces a unique case ID
  • Raw artifacts are stored verbatim
  • Analyses can be replayed and re-inspected
  • No post-hoc modification of evidence

Decisions can be revisited months later with the same evidence.

Why This Is Not “LLM-as-Judge”

  • No model scores itself
  • No subjective rubrics
  • No outcome-based grading
  • No preference voting

The instrument measures structure extracted from outputs, not opinions about quality.

Audit-Grade by Design

  • Immutable artifacts
  • Explicit reasoning topology
  • Cross-model comparability
  • Exportable forensic exhibits

Suitable for governance, compliance, and external review.
