How it works: a replayable measurement pipeline for structural support in AI reasoning

Reflective Diagnostics measures whether an AI system’s conclusions are structurally supported by its stated premises. It does not grade quality, truth, or preference. It produces a replayable evidence record suitable for audit and external review.

End-to-End Measurement Flow

Input→ Controlled Run→ Artifact Capture→ Structural Extraction→ Signal Computation→ Interpreter→ Evidence Packet

No hidden rewrites. No silent retries. No adaptive steering.

Controlled Run

  • Same input and constraints across models
  • Model identity and version recorded
  • No retries or prompt rewriting

Artifact Capture

  • Raw model outputs stored verbatim
  • Run metadata captured
  • Hashes prevent silent alteration

Structural Extraction

  • Claims, premises, conclusions identified
  • Support relationships made visible
  • Orphaned conclusions exposed

Signal Computation

  • Coverage
  • Orphan rate
  • Stability
  • Integrity

Interpreter

  • Cross-model synthesis
  • Divergence and consistency surfaced
  • Failure modes tied to artifacts

Evidence Packet

  • Replayable forensic record
  • Exportable audit exhibit
  • Designed for governance and funding review
