How it works: a replayable measurement pipeline for structural support.

Reflective Diagnostics converts AI output into an inspectable support structure (claims → premises → conclusions), computes structural signals (coverage, orphans, stability, integrity), and produces a replayable evidence record suitable for audit and external review. Non-goal: truth verification.

END-TO-END MEASUREMENT FLOW

Outputs from each stage become fixed inputs to the next stage. The instrument is designed to prevent hidden changes that would break auditability.

No hidden rewrites. No silent retries. No adaptive steering.

1) CONTROLLED RUN (COMPARABILITY)

  • Same input, same constraints, across models
  • Model identity and version recorded
  • No prompt rewriting or hidden retries
Differences observed are attributable to the model, not shifting conditions.

2) ARTIFACT CAPTURE (SOURCE OF RECORD)

  • Raw outputs stored verbatim
  • Run metadata captured (timestamps, run ID, model selection)
  • Hashes generated to prevent silent alteration
A run can be audited by someone who did not witness it.

3) STRUCTURAL EXTRACTION (SUPPORT GRAPH)

  • Claims, premises, and conclusions explicitly identified
  • Support relationships made visible
  • Orphaned (unsupported) conclusions surfaced
Narrative output becomes inspectable structure.

4) SIGNAL COMPUTATION (STRUCTURAL METRICS)

  • Coverage: how much of the conclusion is supported
  • Orphan rate: unsupported commitments
  • Stability: consistency under controlled variation
  • Integrity: internal coherence of the support chain
Signals are derived from structure — not preference or outcome grading.

5) INTERPRETER (CROSS-MODEL SYNTHESIS)

  • Cross-model comparison under identical constraints
  • Divergence and consistency made explicit
  • Failure modes shown with supporting artifacts
This is how evaluation becomes decision-grade, not subjective.

6) EVIDENCE PACKET (EXPORTABLE EXHIBIT)

  • Replayable record: inputs, outputs, and computed signals
  • Exportable forensic exhibit format
  • Designed for audit, governance, and funding review
The output is not a score — it’s a case file.
